Troubleshooting

If you cannot start Rawr or frequently run into Crashes/Errors, this is the page for you.

>>ISSUES SHOULD BE POSTED IN THE ISSUE TRACKER PER THE POSTING GUIDELINES<<

These Issues are preventing people from running Rawr4, follow the advice and you won't see the same problems.

THE MOST COMMON ISSUE IS USING AN OUT OF DATE SILVERLIGHT (AKA SILVERLIGHT 3 OR EARLIER) YOU NEED SILVERLIGHT 4.

  1. Windows update may not be enough to get you the current Silverlight version, which is what we are using for Rawr4.
    - Go to the Silverlight download page so you can be prompted to get the latest version: http://www.microsoft.com/getsilverlight/Get-Started/Install/Default.aspx
    - FIXED: You should be getting appropriately popped for SL4 now
  2. Cannot Launch Rawr because of a Parser Error
    - See #1
  3. Cannot Launch Rawr because of an IsolatedStorage_Operation_ISFS
    - You denied Rawr the ability to make a Cache/Use Storage. Go into Silverlight settings and allow it.
  4. Cannot Launch Rawr because the ItemCache loads in and breaks due to HolyShield being a bad Trigger (or some other similar nonsense)
    - You need to delete your Rawr3 saved caches, Right-click the window, choose Silverlight go to the last tab (named Application Storage) and select elitistjerks.com then click the Delete button, then Relaunch Rawr (or refresh the browser page)
  5. You got into Rawr but there are no Items present
    - Your ItemCache didn't load for whatever reason. Go to Tools > Reset All Caches to redownload the new data
    - DPSDK also had this issue specifically, this was a filtering bug and has been fixed for 4.0.11 (current release 4.0.10)
  6. Install Offline button doesn't work
    - You probably already installed it offline, right click anywhere and it will tell you "Remove Application" in the menu if you have
  7. Google Chrome Users, it looks like you need to force the browser to refresh the page instead of running the cached version using Shift+F5. Otherwise you will see the old version and that can cause other crashes since new data files have been posted.

Common Preventions of Rawr2's Launch

The application failed to initialize properly (0xc0000135)
If you get a 'The application failed to initialize properly (0xc0000135)' or similar error when starting Rawr you most likely don't have .NET Framework installed. You can get it from http://www.microsoft.com/NET/.

Cannot access a disposed object
If you get a 'Cannot access a disposed object' error, that's a bug in an old version of the .NET Framework. Uninstall all versions of .NET Framework that you have, and reinstall the latest from Microsoft (3.5SP1)

Could not load file or assembly Rawr.Base
You have to fully unzip Rawr from the zip file.

Using Rawr 2.x with Mono
If you're running Rawr via Mono on OSX/Linux, you'll need to use Mono v2.2. The latest version (v2.4) has a bug that prevents Rawr from running. Some users have reported that the issues with 2.4 have been resolved, however others still report crashing.

Using Rawr 4 with Mono/Moonlight
Unfortunately, the Silverlight port to Linux (Moonlight) will not be able to support the Silverlight 3 protocols in time for Rawr3. At the time of Rawr3's full release, Rawr 2 will be retired and, unless a drastic change occurs with Moonlight, Linux Users will no longer be able to run Rawr.

TalentPicker.cs has Thrown an Exception
You may see this if you are running a Mac OS and trying to launch Rawr2 with Parallels. Try running it with bootcamp instead.

Troubleshooting Questions: Launching Rawr

  1. Did you recently update to the current version (unless told otherwise for specific model use)?
    1. Try going back a version and see how it behaves differently
  2. Did you recently perform a Windows Update or install any major component on your system (Drivers, etc.)?
    1. Try rolling back using the Windows System Restore setting
  3. Did you install the correct version of .NET for your Operating System?
    1. Try searching for the installer that best matches your OS, taking note of the Version (3.5 SP1), type (x86 or x64) and OS it's for (XP, Vista, 7)
  4. Did you install Rawr using the Installation Instructions to a directory where you have full access rights, such as My Documents or the Desktop?
    1. We don't recommend installing Rawr to the Program Files directory as on many systems, even if you do have access rights, it won't have the ability to call and edit it's files where necessary.
    2. We don't recommend extracting a new version of Rawr onto the same directory as a previous version without deleting the original. Several files may get their wires crossed if you do and that can cause crashing.
  5. Have you run a Virus/Malware Scan lately?
    1. It's entirely possible that a Virus/Malware program is preventing Rawr (and other programs) from launching
  6. Are you running an Internet Security Suite, such as AVG or Norton?
    1. Security Suites have been known to flag some programs inappropriately as Malware and prevent them from launching. Check with your Suite's Help documentation to see if there's a way to add Rawr as an Exception if this is the case.

Troubleshooting Questions: Character Armory Imports

  1. Did it work in your previous version?
    1. Check the settings you had in the previous version and try to match. It is possible to Run multiple versions of Rawr at the same time (but not multiple of the same version).
  2. Are you using a Proxy (as many Corporate Environments do)?
    1. Did you enter the proxy info into Rawr (Tools > Options > Network Settings)
    2. Are you using a Domain and did you enter DOMAINNAME\username vs just username? If you did, try the other method
  3. Is your Region's Armory down for Maintenance?
  4. Is the US's Armory down for Maintenance (even though you are using a non-US region server)?
  5. Is it erroring on a specific item?
    1. Many tabards and shirts cause issues as they are not posted to the Armory as items (and therefore can't be called by Rawr). We have tried to work around this as much as possible but new items do get added and we don't know there's a problem until it happens
  6. Does your Toon have any special characters in it's name?
    1. This is a translation issue when talking to the Armory, you may need to put the ASCII Character Code in instead of the character itself to get it to work
  7. Have you added exceptions for Rawr to ALL of your Firewalls (including Windows)?
  8. Try the following: Go to Tools > Options > NetworkSettings, and change User Agent from "Internet Explorer 6" to "FireFox 2".
  9. Make sure Rawr internet access is not blocked by any other programs. From one of our users that resolved the issue: "At one time in the recent past I had a trial of a norton product on my computer and uninstalled it.. However it would appear that not all of it was removed so there is a tool you can download from symantec that is called norton removal tool. I ran that and rebooted and i was able to use rawr to import from armory again."
  10. Is the Armory up but not able to display your character per the reasons listed below (taken from an armory error message)
    1. The character has been inactive for an extended period of time.
    2. The character name was spelled incorrectly.
    3. The profile is temporarily unavailable while the character is in the midst of a process such as a realm transfer or faction change.
    4. Characters that have been deleted are no longer available on the Armory.


If/when you receive an error on the Armory Load status pane, you can double-click the error listing and it will open a box with more details. Use the Copy to Clipboard button for providing us the error message in a new Issue per the Posting Guidelines.

In addition we might need additional logging information to assist in the issue. To generate a System.Net log follow these steps:

Open Rawr.exe.config file in a text editor (i.e. notepad). Insert the following before the last line of the document (so it is nested inside the configuration element):

<system.diagnostics>
    <sources>
        <source name="System.Net" switchValue="Verbose">
            <listeners>
                <add name="System.Net"/>
            </listeners>
        </source>
        <source name="System.Net.Sockets" switchValue="Verbose">
            <listeners>
                <add name="System.Net"/>
            </listeners>
        </source>
    </sources>
    <sharedListeners>
        <add name="System.Net" type="System.Diagnostics.TextWriterTraceListener" initializeData="System.Net.log"/>
    </sharedListeners>
    <trace autoflush="true" />
</system.diagnostics>


and then start Rawr again and try to open character from armory.

Attach the generated System.Net.log file in the new issue together with the error message.
